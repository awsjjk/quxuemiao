Section 3: API 路由 & 数据流

Flask RESTful API — JWT 认证 + 4 大业务模块

API 端点总览

方法 路径 说明 认证
Auth — /api/auth
POST /api/auth/login 用户登录，返回 JWT 否
POST /api/auth/register 用户注册 (家长/家教) 否
GET /api/auth/user_info 获取当前用户信息 JWT
PUT /api/auth/update_profile 更新个人信息 + 家长/家教详情 JWT
Demand — /api/demand
POST /api/demand/create 发布家教需求 JWT
GET /api/demand/list 我的需求列表 (家长视角) JWT
GET /api/demand/:id 需求详情 JWT
PUT /api/demand/:id 修改 / 取消需求 JWT
AI Match — /api/match
POST /api/match/run 对指定需求执行 AI 匹配 JWT
GET /api/match/result/:demand_id 获取匹配结果 JWT
Order — /api/order
POST /api/order/create 家长从匹配结果中选择家教，创建订单 JWT
GET /api/order/list 我的订单列表 (家长/家教双视角) JWT
GET /api/order/:id 订单详情 JWT
PUT /api/order/:id/status 更新订单状态 (进行中 → 已完成) JWT
Rating — /api/rating
POST /api/rating/submit 提交服务评价 JWT
GET /api/rating/tutor/:tutor_id 查看家教评价历史 JWT

核心场景数据流: 需求发布 → AI 匹配 → 下单

1. POST /api/demand/create → 写入 demand_info 表
2. POST /api/match/run → Flask 调用 ai_module.agent.match()
   2a. agent 从 MySQL 读取 demand + tutors
   2b. agent 调用 rag.retrieve() → ChromaDB 向量检索
   2c. agent 调用 llm_client.chat(prompt) → 打分 JSON
   2d. agent 返回 [(tutor, score, reason), ...]
3. GET /api/match/result/:id → 前端展示 Top-5
4. POST /api/order/create → 写入 order_info，更新 demand 状态

认证方案

沿用现有 Flask-JWT-Extended 方案：登录返回 Token → 前端存 localStorage → 每次请求带 Authorization: Bearer <token> → 后端 @jwt_required() 校验
