Section 2: AI 匹配模块 (RAG + Agent)

规则引擎初筛 → RAG 知识检索 → LLM 智能打分排序

匹配流程
Step 1
规则引擎初筛 学科 + 年级 + 区域 + 时间 + 预算 → 从 MySQL 过滤出候选家教池 (N≤50)
↓
Step 2
RAG 知识库检索 向量检索候选家教资质 + 该学科知识点 → 构造 LLM 上下文
↓
Step 3
LLM 智能打分 Prompt = 需求描述 + 候选家教(RAG增强) + 学科知识点 → LLM 打分 + 推荐理由
↓
Output
Top-5 推荐列表 每个推荐包含：得分 + 匹配理由 + 家教信息摘要 → 返回前端展示

ai_module/ 内部结构

ai_module/
├── config.yaml ← api_key / model / endpoint
├── __init__.py
├── llm_client.py ← 统一 LLM 调用接口 (适配多平台)
│    class LLMClient: chat(prompt) → response
├── embedding.py ← 文本向量化 (家教资料/需求/知识点)
│    def embed(text) → vector
├── rag.py ← 向量检索 + 知识库管理
│    class RAGRetriever: search(query, top_k) → docs
│    class KnowledgeBase: add/update/delete 知识条目
├── agent.py ← 匹配决策 Agent
│    class MatchAgent: match(demand, candidates) → [(tutor, score, reason)]
└── data/ ← 知识库数据
    ├── subjects.json ← 学科知识图谱
    └── prompts/ ← Prompt 模板
        └── match_scoring.txt

关键设计决策

向量存储 使用 numpy + faiss (轻量) 或 ChromaDB，不引入独立向量数据库
Embedding 复用 LLM API 的 embedding 接口 (如 text-embedding-3-small)
RAG 知识源 家教资质 (skills/certificates/experience) + 学科知识点 (各年级大纲)
LLM 输出格式 JSON 结构化输出: [{tutor_id, score, reason}]，便于前端渲染
